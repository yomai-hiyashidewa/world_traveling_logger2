# 実装順序

TDDで進めるための実装フェーズと順序。
各フェーズは「テストを書く → 実装する → リファクタ」のサイクルで進める。

## Phase 1: プロジェクト基盤

### 1-1: プロジェクト初期化
- Vite + React + TypeScript プロジェクト作成
- Vitest 設定
- ESLint + Prettier 設定
- ディレクトリ構成作成
- 最初のテストが通ることを確認

### 1-2: CI/CD
- GitHub Actions ワークフロー作成
- テスト実行 → ビルド → GitHub Pages デプロイ
- CSVファイルのコピースクリプト

## Phase 2: ドメイン層 — モデル（TDD）

### 2-1: 列挙型定義
- CountryType（220+国）
- CurrencyType（50+通貨）
- AccommodationType, Transportationtype, SightseeigType, OtherType, PlaceType
- 文字列↔列挙値の相互変換関数
- **テスト**: 全列挙値の変換テスト

### 2-2: 基本エンティティ
- IContext インターフェース
- Accommodation モデル
- Transportation モデル（派生プロパティ含む）
- Sightseeing モデル（キーワード自動分類含む）
- OtherExpense モデル
- ExchangeRate モデル
- **テスト**: 生成・プロパティ・派生値テスト

### 2-3: 集計モデル
- CostSummary
- TypeSummary / TransportationTypeSummary
- MovingSummary
- 通貨フォーマッター
- **テスト**: 集計計算・フォーマットテスト

## Phase 3: ドメイン層 — パーサー（TDD）

### 3-1: CSVリーダー
- 汎用CSVパース（カンマ分割、行分割）
- **テスト**: 正常CSV・空行・特殊文字

### 3-2: 各カテゴリパーサー
- AccommodationParser（行→Accommodation変換）
- TransportationParser
- SightseeingParser（タイプ自動分類 + 経費振り分け）
- ExchangeRateParser
- バリデーション + エラー収集
- **テスト**: 正常行・不正行・バリデーションエラー

## Phase 4: ドメイン層 — サービス（TDD）

### 4-1: ExchangeRateService
- レート取得（通貨+日付）
- 平均レート算出
- 通貨変換（price → JPY/EUR/USD）
- **テスト**: レート取得・変換精度・存在しない日付

### 4-2: FilterService
- 表示モードフィルタ（World/Country/Region）
- 日付範囲フィルタ
- 交通フィルタ（飛行機・国境越え）
- 経費フィルタ（保険）
- 国・地域リスト生成
- **テスト**: 各フィルタ単体・複合フィルタ

### 4-3: StatsService
- カテゴリ別費用集計
- タイプ別集計
- 移動メトリクス集計
- 旅行概要（国数・日数等）
- **テスト**: 集計計算全パターン

### 4-4: RouteService
- 入国・出国判定
- 地域間移動分析
- NO_ENTRY処理
- 訪問国リスト生成
- **テスト**: ルート判定全パターン

## Phase 5: UI層 — 状態管理

### 5-1: データ読み込みフック
- useTravelData: CSV fetch → パース → 通貨変換 → State保持
- ローディング・エラー状態管理
- **テスト**: フック統合テスト

### 5-2: フィルタ状態フック
- useFilterState: フィルタ条件管理
- フィルタ変更→再計算のトリガー
- **テスト**: 状態遷移テスト

### 5-3: 集計フック
- useStats: フィルタ済みデータ→集計結果
- **テスト**: 統合テスト

## Phase 6: UI層 — コンポーネント

### 6-1: 基盤コンポーネント
- DataTable（ソート可能テーブル）
- StatsSummary（集計表示）
- LoadingIndicator

### 6-2: フィルタ系コンポーネント
- FilterPanel（トグル・日付ピッカー・通貨セレクタ）
- Sidebar（国・地域リスト）

### 6-3: タブ系コンポーネント
- AccommodationTab
- TransportationTab
- SightseeingTab
- OtherTab
- RouteTab

### 6-4: ページ統合
- Dashboard（全コンポーネント統合）
- App（ルーティング不要、単一ページ）

## Phase 7: 仕上げ

### 7-1: スタイリング
- レスポンシブ対応
- 元アプリのレイアウト再現

### 7-2: 互換性テスト
- 元アプリのCSVサンプルデータでの動作確認
- 集計結果の一致検証

### 7-3: デプロイ
- GitHub Pages への初回デプロイ
- 動作確認
