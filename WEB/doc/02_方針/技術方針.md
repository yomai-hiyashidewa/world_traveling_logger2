# 技術方針

## デプロイ先

- **GitHub Pages**（静的ホスティング）
- バックエンドサーバーなし
- データベースなし
- 全ロジックはクライアントサイドで完結

## データソース

- `documents/English/` 以下のCSVファイルを起動時に `fetch` で取得
  - `accommodations.csv`
  - `transportations.csv`
  - `sightseeing.csv`
  - `exchange_rates.csv`
- CSVはリポジトリに同梱し、ビルド時に公開ディレクトリへコピー

## 画像ソース（FR-011）

- `documents/image/` 以下の画像ファイルをビルド時に `public/image/` へコピー
  - `Flags/{CountryCode}.png` — 国旗画像（83枚）
  - `Countries/{CountryCode}/zero.jpg` — 国別メイン写真（77カ国）
  - `world.JPEG` — ワールドモード画像
  - `Icon/icon.png` — favicon
- 元WPFアプリの OptionModel.ImagePath に相当するが、WEB版ではパス固定（設定不要）

## アーキテクチャ原則

### 1. Domain層はReactに依存しない

- `src/domain/` 配下は純粋なTypeScript
- import に `react` を含めない
- これにより Vitest で高速にテスト可能
- 元WPFの Model層 がそのまま対応

### 2. UI層はDomain層を利用するだけ

- React コンポーネントは表示とイベントハンドリングに専念
- ビジネスロジックは hooks 経由で domain/ の service を呼ぶ
- コンポーネント内に計算ロジックを書かない

### 3. 依存の方向

```
ui/ → hooks/ → domain/services/ → domain/models/
                                 → domain/parsers/
```

- domain は ui に依存しない（逆方向の依存禁止）

## TDD 戦略

### テストの優先順位

1. **domain/models/** — エンティティ生成・列挙型変換・通貨変換
2. **domain/parsers/** — CSVパース・バリデーション・エラー検出
3. **domain/services/** — フィルタ・集計・ルート分析
4. **ui/hooks/** — 状態管理の統合テスト
5. **ui/components/** — コンポーネントの表示テスト

### テスト実行

```bash
# ドメイン層のみ（高速、TDDサイクル中に常時実行）
npx vitest run tests/domain/

# 全テスト
npx vitest run

# ウォッチモード（TDD時）
npx vitest --watch
```

### テストカバレッジ目標

| レイヤー | カバレッジ目標 |
|---|---|
| domain/models | 90%以上 |
| domain/parsers | 90%以上 |
| domain/services | 85%以上 |
| ui/hooks | 70%以上 |
| ui/components | 主要操作のみ |

## データフロー

```
起動
  ↓
fetch("./data/*.csv")
  ↓
domain/parsers/ でパース・バリデーション
  ↓
domain/services/exchangeRateService で通貨変換
  ↓
React State に全データ保持
  ↓
ユーザー操作（フィルタ変更）
  ↓
domain/services/filterService でフィルタ適用
  ↓
domain/services/statsService で集計計算
  ↓
React コンポーネントが再レンダリング
```

## ビルド・デプロイ

### ローカル開発

```bash
npm install
npm run dev          # Vite 開発サーバー（HMR付き）
npm run test         # Vitest 実行
npm run test:watch   # TDD ウォッチモード
npm run build        # 本番ビルド
npm run preview      # ビルド結果プレビュー
```

### GitHub Pages デプロイ

- `main` ブランチへの push で GitHub Actions が自動実行
- `npm run build` → `dist/` を GitHub Pages へデプロイ
- ベースパス: `/{repository-name}/` を vite.config.ts で設定

### CSVファイルの配置

- ビルド時に `documents/English/*.csv` を `public/data/` へコピー
- vite.config.ts のプラグインまたは npm script で自動化

### 画像ファイルの配置

- ビルド時に `documents/image/` を `public/image/` へ再帰コピー
- vite.config.ts の copyCsvPlugin を拡張し、画像コピーも自動化
- 対象: Flags/, Countries/, Icon/, world.JPEG 等
