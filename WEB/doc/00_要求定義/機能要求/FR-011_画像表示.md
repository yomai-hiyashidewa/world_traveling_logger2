# FR-011: 画像表示

## 概要

元WPFアプリで実装されていた国旗・国別写真の表示機能をWEB版でも提供する。
画像は `documents/image/` フォルダに格納されており、ビルド時に公開ディレクトリへコピーして使用する。

元コード: OptionModel.ImagePath, CountryViewModel.ImagePath, SideViewModel.CountryFlagPath / CountryImagePath

## 画像フォルダ構成

```
documents/image/
├── Flags/                  ... 国旗画像（PNG）
│   ├── JPN.png
│   ├── AUS.png
│   ├── USA.png
│   └── ... (83枚)
├── Countries/              ... 国別メイン写真（JPG）
│   ├── JPN/
│   │   └── zero.jpg
│   ├── AUS/
│   │   └── zero.jpg
│   └── ... (77カ国)
├── Icon/                   ... アプリアイコン
│   ├── icon.png
│   └── icon.bmp
├── world.JPEG              ... ワールドモード時のメイン画像
├── japan.jpg
├── australia.jpg
└── ... (地域別画像)
```

## 要求一覧

### FR-011-01: 国旗表示

- 国リスト・国選択ドロップダウン・ルート表示において、国コードに対応する国旗画像を表示する
- 画像パス: `image/Flags/{CountryCode}.png`（例: `image/Flags/JPN.png`）
- 元コード: CountryViewModel.ImagePath → `Path.Combine(imageDir_, "Flags", countryType_.ToString() + ".png")`
- 該当する国旗ファイルが存在しない場合は国旗を表示しない（フォールバック）

### FR-011-02: 国別メイン写真表示

- サイドバーまたはヘッダー領域に、現在の表示モードに応じた画像を表示する
  - **ワールドモード**: `image/world.JPEG` を表示
  - **カントリーモード**: `image/Countries/{CountryCode}/zero.jpg` を表示（例: `image/Countries/AUS/zero.jpg`）
- 元コード: SideViewModel.CountryImagePath
- 該当する画像ファイルが存在しない場合はワールド画像にフォールバック

### FR-011-03: ルート表示における国旗

- 国境越え一覧の出発国・到着国にそれぞれ国旗アイコンを表示する
- 元コード: RouteRegionViewModel.CountryFlagPath → CountryViewModel.ImagePath

### FR-011-04: アプリアイコン

- WEB版の favicon として `image/Icon/icon.png` を使用する

### FR-011-05: 画像ファイルの配置

- ビルド時に `documents/image/` を `public/image/` へコピーし、GitHub Pages で配信する
- WEB版では画像パスは固定（元WPFの OptionModel.ImagePath による動的設定は不要）

## テスト観点

- 国旗画像が存在する国コードで正しく表示されること
- 国旗画像が存在しない国コードでエラーにならず、フォールバック動作すること
- ワールドモード時にワールド画像が表示されること
- カントリーモード切替時に該当国の写真に切り替わること
- 該当国の写真が存在しない場合にワールド画像にフォールバックすること
- ルート表示の各国境越えに国旗が表示されること
- favicon が正しく設定されること
