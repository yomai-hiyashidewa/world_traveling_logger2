# FR-002: 宿泊記録管理

## 概要

旅行中の宿泊に関する記録を管理する。

## 要求一覧

### FR-002-01: 宿泊記録の登録

- 以下の項目で宿泊記録を登録できる
  - 日付（必須）
  - 国（必須、CountryType列挙値）
  - 地域（任意）
  - 宿泊タイプ（必須、AccommodationType列挙値）
  - 料金（必須、数値）
  - 通貨（必須、CurrencyType列挙値）
  - メモ（任意）

### FR-002-02: 宿泊タイプ分類

- 以下のタイプに分類される:
  - ParentsHouse, Domitory, SingleRoom, Airplane, Bus, Ferry, Train, Airport, FriendHouse, Hotel

### FR-002-03: 宿泊記録の一覧表示

- フィルタ条件に基づいた宿泊記録の一覧を表示できる
- 一覧はDataGrid形式で表示
- 国カラムには国コード（JPN等）ではなく**国名（Japan等）**を表示する
  - 元コード CountryType.cs の `[Display(Name=...)]` 属性と同等のマッピングを使用
  - `countryName()` 関数で変換（約200カ国対応）

### FR-002-04: 宿泊タイプ別集計

- 宿泊タイプごとの件数・合計費用・最大・最小・平均費用を集計表示できる

## テスト観点

- 全必須フィールドが揃った登録が成功すること
- 必須フィールド欠如時にバリデーションエラーとなること
- 宿泊タイプ列挙値の変換が正しいこと
- タイプ別集計値（件数・合計・最大・最小・平均）が正しいこと
- フィルタ条件適用後の一覧が正しいこと
