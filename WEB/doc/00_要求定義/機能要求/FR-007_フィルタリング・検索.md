# FR-007: フィルタリング・検索

## 概要

旅行記録を多様な条件で絞り込み、表示対象を制御する。

## 要求一覧

### FR-007-01: 表示モード切替

- **ワールドモード**: 全世界のデータを表示
- **カントリーモード**: 特定国のデータのみ表示
- **リージョンモード**: 特定国の特定地域のデータのみ表示

### FR-007-02: 日付範囲フィルタ

- 開始日・終了日を指定してデータを絞り込める
- デフォルトはデータ全期間

### FR-007-03: 国・地域フィルタ

- 国を選択してフィルタ（CountryType列挙値）
- 国選択ドロップダウンには国コードではなく**国名（Display Name）**を表示する
  - 国名変換は `countryName()` 関数により CountryType.cs の Display属性と同等のマッピングで行う
- 国内の地域を選択してさらに絞り込み

### FR-007-04: 交通フィルタ

- **飛行機移動の除外**: チェックボックスで AirPlane タイプの交通記録を包含/除外切り替え
  - 元コード: ControlModel.IsWithAirplane に相当
  - 除外時、交通タブ・集計・ルート分析すべてから AirPlane レコードが除かれる
- **国境越え移動の除外**: チェックボックスで国境越え（startCountry ≠ endCountry）の交通記録を除外
  - 元コード: ControlModel.IsWithCrossBorder に相当
  - **カントリーモード時のみ有効**（ワールドモード時は非表示・無効）
  - 除外時、出発国と到着国が異なる交通レコードがフィルタされる

### FR-007-05: 経費フィルタ

- **保険費用の除外**: チェックボックスで Insurance タイプのその他経費を包含/除外切り替え
  - 元コード: ControlModel.IsWithInsurance に相当
  - 除外時、その他経費タブ・集計から Insurance レコードが除かれる

### FR-007-06: 日本除外フィルタ

- ワールドモードで日本のデータを除外する切り替え

### FR-007-07: フィルタ適用時の再計算

- フィルタ条件変更時、集計・統計・表示が即座に再計算される
- ControlChangedイベントに相当するリアクティブな更新
- WEB版では React の状態変更 → useMemo による再計算で実現

### FR-007-08: フィルタクリア

- 「Clear」ボタンで全フィルタ条件をリセットできる
- 国・地域選択、日付範囲、各トグル（飛行機・保険・国境越え）がすべて初期値に戻る

## テスト観点

- 各モード切替で表示データが正しく絞り込まれること
- 日付範囲フィルタの境界値（開始日を含む/終了日を含む）
- 国フィルタ適用後の地域リスト更新
- 飛行機除外トグル: ON時に AirPlane レコードが全タブ・集計から消えること
- 保険除外トグル: ON時に Insurance レコードがその他経費タブ・集計から消えること
- 国境越え除外トグル: カントリーモード時のみ表示されること、ON時に startCountry ≠ endCountry のレコードが除外されること
- 日本除外フィルタの動作
- 複合フィルタ（日付+国+交通タイプ+トグル）の組み合わせ
- Clearボタンで全フィルタがリセットされること
- フィルタ変更後の再計算トリガー
