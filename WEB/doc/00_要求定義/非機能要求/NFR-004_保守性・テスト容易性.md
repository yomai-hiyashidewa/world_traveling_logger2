# NFR-004: 保守性・テスト容易性

## 概要

TDDで開発し、長期的に保守可能なコードベースを維持する。

## 要求一覧

### NFR-004-01: テスト容易性

- ビジネスロジックはUI・インフラ層から独立してテスト可能
- 依存関係はインターフェース経由で注入（DI）
- 各仕様項目に対応するユニットテストが書ける粒度

### NFR-004-02: アーキテクチャ

- レイヤードアーキテクチャ（プレゼンテーション / アプリケーション / ドメイン / インフラ）
- ドメインロジックは外部依存なし（Pure Function推奨）
- フロントエンド・バックエンドの責務分離

### NFR-004-03: コード品質

- 型安全性（TypeScript / 型付き言語の使用）
- Linter/Formatterの導入
- CI/CDパイプラインでのテスト自動実行

### NFR-004-04: ドキュメント

- APIドキュメント（OpenAPI/Swagger等）の自動生成
- コンポーネントカタログ（Storybook等）の検討

## テスト観点

- ユニットテストカバレッジ: ドメイン層80%以上
- インテグレーションテスト: API層の全エンドポイント
- E2Eテスト: 主要ユーザーフロー
